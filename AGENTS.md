# AGENTS.md

## 核心原则：正常环境默认
1. 默认用户网络可用，且未配置显式代理环境变量。
2. 默认本地服务通过常见方式访问（如 `localhost`、常见本地域名、容器内网域名）。
3. 默认用户会使用常见排查手段验证可达性（如 `ping`、`curl`、`nslookup`）。
4. 项目内策略（如 `fake-ip`）应优先保障上述常见方式可用。

## 职责边界
1. 只处理项目可直接影响的内容。
2. 不把外部环境异常写成项目默认前提或默认配置要求。
3. 仅在用户明确提供证据或明确要求时，才将外部环境问题纳入主结论。

## 表达约束
1. 事实与推断分离：先给已验证事实，再给推断。
2. 非必需项必须条件化表达：使用“仅当 X 时才需要 Y”。
3. 用户纠偏后，不重复使用被否定的前提。
